🚀 Vision – Natural Data Engine NDE • PHP 8.3+

⚡ Zéro Configuration (Plug & Play)

Contrairement aux bases SQL (MySQL, PostgreSQL) ou NoSQL (MongoDB), Vision ne nécessite aucune installation de serveur, aucun compte utilisateur, et aucune configuration.

Le simple fait d'inclure vision.php dans votre code active instantanément toute l'infrastructure (Base de données, Cache, Verrouillage, Worker).

Vision est un moteur de données naturel (NDE : Natural Data Engine).
Un NDE est une évolution des bases de données traditionnelles : il stocke, lit et relie des données comme une base classique, mais il le fait sans schéma, sans tables, sans SQL et sans structure imposée.

Vision repose sur deux fondations simples : UniCell, où chaque donnée est une cellule autonome stockée dans un fichier .vr, et Wave, un état global minimal qui coordonne le moteur.

Contrairement aux bases de données classiques, Vision ne prédit rien, ne cascade rien automatiquement et ne restructure jamais vos données. Il vous laisse le contrôle total, tout en garantissant une stabilité et une prévisibilité maximales.

1. Introduction

Vision est un moteur de données artisanal en PHP 8.3+, UTF‑8 LF, sans dépendances. Il est autonome, simple, stable et prévisible. Il reconstruit naturellement des structures complexes : forums, CMS, knowledge graphs, présentations, données scientifiques, etc.

  • UniCell – chaque donnée est une cellule autonome stockée dans un fichier .vr. Les relations utilisent des niveaux simples :
    • 0 = latéral
    • 1 = interne directe
    • 2 = interne profonde
    • 3+ = cas rares
  • Wave – un état global minimal, mis à jour en continu. Chaque module lit Wave et réagit localement. Aucune cascade. Aucune prédiction.

3. UniCell – format .vr

Chaque cellule est stockée dans un fichier .vr :

Format .vr 
VISION1
⟪¦<base64(json)>¦⟫
⟪⇒¦⇐⟫

Le JSON interne contient :

JSON interne 
{
  "id": "20260416_ab12cd34ef56",
  "data": {...},
  "relations": [
      {"id": "xxx", "niveau": 1},
      {"id": "yyy", "niveau": 0}
  ],
  "createdAt": 1713200000,
  "updatedAt": 1713200000
}
  • atomicité stricte : .tmp → flush → rename()
  • aucune écriture partielle
  • validation par marqueur final
  • format déterministe
  • UTF‑8 LF

4. Sharding

Chaque cellule est stockée dans :

Sharding par date 
/vision/data/YYYYMMDD/id.vr

Le préfixe de l’ID (YYYYMMDD) détermine automatiquement le shard. Les tests multi‑shards garantissent que les relations, graphes et suppressions en cascade restent cohérents sur plusieurs jours.

5. Wave – état global

Wave est un tableau associatif simple :

Structure de Wave 
[
  'charge'   => float,   // 0.0 → 1.0
  'priorite' => string,  // read | write | clean | idle
  'pression' => string,  // low | medium | high
  'lock'     => string,  // free | busy
  'ram'      => string,  // stable | high | critical
  'disk'     => string,  // idle | busy
  'op'       => string   // type d’opération
]

Chaque module lit Wave et réagit localement. Aucune orchestration lourde, aucune prédiction, aucune cascade globale.

7. API Vision (complète)

L’API Vision est exposée via Vision_API.php. Les fonctions suivantes sont disponibles en mode statique (ex. Vision_API::create(...)).

Fonction Description Signature
create Créer une cellule create(array $data, array $relations = []): string
read Lire une cellule read(string $id): ?array
update Mettre à jour une cellule update(string $id, array $data, array $relations = []): bool
delete Supprimer une cellule delete(string $id): bool
exists Vérifier l’existence d’une cellule exists(string $id): bool
relations Lire les relations d’une cellule relations(string $id): array
link Créer une relation link(string $id, string $targetId, int $niveau): bool
unlink Supprimer une relation unlink(string $id, string $targetId): bool
graph Construire un graphe autour d’une cellule graph(string $id): array
find Recherche personnalisée (filtre PHP) find(callable $filter): array
count Nombre total de cellules count(): int
stats Statistiques globales (taille, shards, etc.) stats(): array
clean Maintenance explicite clean(): void

  • triggers
  • prédictions
  • réorganisation automatique globale
  • analyse du contenu
  • inférence de schéma
  • hiérarchie forcée

Vision reste volontairement minimaliste et prévisible. Toute logique métier, tout modèle de données et toute hiérarchie restent sous votre contrôle.

11. Pourquoi Vision ?

Vision n’est pas une base SQL, ni un document store, ni une base orientée graphe classique. C’est un moteur de données naturel (NDE) conçu pour offrir une alternative simple, robuste et prévisible aux systèmes traditionnels.

Comparé à une base SQL

  • aucune table, aucun schéma, aucune migration
  • aucune jointure complexe
  • stockage direct en fichiers
  • relations simples et explicites
  • lecture ultra‑rapide (RAM‑first)

Comparé à un document store

  • chaque cellule est autonome (UniCell)
  • relations natives (pas d’embed arbitraire)
  • sharding naturel par date
  • format déterministe et auto‑validé

Comparé à une base orientée graphe

  • relations simples (niveau 0, 1, 2…)
  • navigation multi‑shards
  • résistance aux cycles
  • aucune réécriture globale du graphe

Ce que Vision apporte

  • un moteur ultra‑simple à comprendre
  • aucune dépendance, aucun serveur
  • écriture atomique, crash‑safe
  • aucune corruption possible
  • structure lisible par un humain
  • parfait pour forums, CMS, notes, graphes, knowledge bases

12. Tutoriel : Créer un micro-système de news

Ce petit guide vous montre comment utiliser Vision pour gérer des articles et des catégories en quelques lignes de code.

App.php — Tutoriel complet 
<?php
// 1. ACTIVATION (Zéro Config)
// Le simple fait d'inclure ce fichier initialise la DB et le cache.
include "vision/vision.php";

// 2. CRÉATION D'UNE CATÉGORIE
// On stocke des données simples. Vision nous donne un ID unique.
$catId = Vision_API::create(['nom' => 'Technologie', 'couleur' => 'bleu']);
echo "Catégorie créée : $catId \n";

// 3. CRÉATION D'UN ARTICLE LIÉ
// On lie l'article à la catégorie via une relation de niveau 1 (directe).
$newsId = Vision_API::create(
    [
        'titre' => 'Vision v18 en production',
        'corps' => 'Le tutoriel est enfin disponible !',
        'vue'   => 0
    ],
    [['id' => $catId, 'niveau' => 1]] 
);
echo "Article créé : $newsId \n";

// 4. LECTURE ET MISE À JOUR
// On lit la donnée, on modifie le compteur de vues et on enregistre.
$article = Vision_API::read($newsId);
$article['data']['vue']++; 
Vision_API::update($newsId, $article['data']);

// 5. RECHERCHE (Find)
// On cherche tous les articles qui ont plus de 0 vue.
// Le générateur (yield) permet de ne pas saturer la RAM.
$populaires = Vision_API::find(fn($cell) => 
    ($cell['data']['vue'] ?? 0) > 0
);

foreach($populaires as $item) {
    echo "Top News : " . $item['data']['titre'] . "\n";
}

// 6. EXPLORATION DU GRAPHE
// On demande l'article et tout ce qui lui est lié (sa catégorie).
$graphe = Vision_API::graph($newsId);
print_r($graphe);

// 7. STATISTIQUES
// Surveiller la taille de la base de données.
$stats = Vision_API::stats();
echo "Total cellules : " . $stats['count'];
