🚀 Vroom : la base de données PHP réinventée

Vroom est une base de données artisanale en PHP, pensée pour être simple, lisible, rapide et incassable. Elle repose sur un principe clair : un enregistrement = un fichier, avec un moteur RAM‑centré, multitâche, qui ne touche le disque qu’en écriture atomique complète.

🧩 Qu’est‑ce que Vroom ?

Vroom est un moteur de stockage minimaliste, conçu pour remplacer une base SQL dans les cas où :

  • la simplicité est prioritaire
  • la performance en lecture/écriture est critique
  • la robustesse et la prévention de corruption sont essentielles
  • le déploiement doit être ultra‑léger

Vroom combine :

  • Vroom BD : stockage physique en fichiers .vrec
  • Vroom Moteur : buffers RAM, transactions, locks, workers, file d’attente, multitâche

📦 Architecture générale

📁 Vroom BD

  • Un fichier .vrec par enregistrement
  • En‑tête ASCII solide (id, type, timestamps, relations)
  • Données encodées en Base64
  • Répartition automatique dans des sous‑dossiers basés sur l’ID
  • Marqueur final ⟪⇒¦⇐⟫ pour vérifier l’intégrité

⚙️ Vroom Moteur

  • Buffers RAM pour chaque fichier ouvert
  • Écriture atomique en une seule passe
  • Transactions RAM → disque
  • Locks physiques simples
  • Workers parallèles (pcntl_fork)
  • File d’attente, supervision, timeouts
  • Cache RAM avec éviction LRU

🧠 Comment fonctionne Vroom ?

1. Lecture

Lorsqu’un fichier est demandé :

  • lecture disque → RAM
  • header + data stockés dans des buffers
  • relations pré‑chargées automatiquement
  • cache RAM mis à jour

2. Modification

Toutes les modifications se font en RAM :

  • aucune écriture partielle
  • aucun fichier ouvert en écriture prolongée
  • aucun risque de corruption

3. Écriture atomique

Lors d’un commit :

  • construction du fichier complet en RAM
  • écriture dans un fichier temporaire
  • rename() atomique
  • marqueur final écrit en dernier

4. Nettoyage automatique

Après chaque écriture :

  • suppression des .tmp orphelins
  • suppression des .lock expirés
  • suppression des .vrec corrompus

⚡ Exemples concrets

Créer un post

<?php
$id=create('post',[
 'title'=>'Bonjour',
 'body'=>'Contenu',
 'user_id'=>1
]);
?>

Lire et modifier

<?php
$p=get('post',$id);
$p['title']='Modifié';
update('post',$id,$p);
?>

Transactions

<?php
begin();
$id1=create('post',['title'=>'A']);
$id2=create('post',['title'=>'B']);
commit();
?>

Workers & file d’attente

<?php
vroom_job_enqueue(function(){
 return ['ok'=>true];
});
vroom_job_run_queue(2);
?>

🧬 Structure interne détaillée

Cette section décrit en profondeur le rôle de chaque fichier interne de Vroom. Elle est destinée aux développeurs, architectes et curieux souhaitant comprendre le fonctionnement exact du moteur.

📌 Vroom.php

Point d’entrée principal. Fournit l’API publique :

  • create(), get(), update(), delete()
  • all(), findBy()
  • begin(), commit(), rollback()
  • info()
  • gestion des jobs (vroom_job_*)

Charge tous les modules internes et orchestre l’ensemble du moteur.

🧠 Vroom_state.php

Le “cerveau” du moteur :

  • état global
  • buffers RAM
  • cache LRU
  • transactions en cours
  • file d’attente
  • workers actifs
  • statistiques internes

🔢 Vroom_id.php

Gestion complète des identifiants :

  • génération d’ID unique
  • incrémentation atomique
  • structure des sous‑dossiers (00/12/34/...)
  • normalisation des chemins

🔒 Vroom_lock.php

Gestion des verrous physiques :

  • création de fichiers .lock
  • expiration automatique
  • prévention des écritures concurrentes
  • nettoyage des locks orphelins

📖 Vroom_disk_read.php

Lecture disque → RAM :

  • lecture sécurisée du fichier
  • extraction du header ASCII
  • décodage Base64
  • vérification du marqueur final ⟪⇒¦⇐⟫
  • mise à jour du cache RAM

✍️ Vroom_disk_do.php

Écriture RAM → disque :

  • construction du fichier complet en RAM
  • écriture dans un .tmp
  • rename() atomique
  • écriture du marqueur final
  • suppression des .tmp orphelins
  • suppression des .vrec corrompus

💾 Vroom_ram.php

Gestion des buffers RAM :

  • stockage temporaire
  • modifications en RAM
  • rollback / commit
  • éviction LRU
  • synchronisation RAM ↔ disque

⚡ Vroom_exec.php

Gestion du multitâche :

  • workers via pcntl_fork()
  • timeouts
  • supervision
  • isolation des jobs

📨 Vroom_job.php

Gestion de la file d’attente :

  • vroom_job_enqueue()
  • vroom_job_run_queue()
  • exécution séquentielle ou parallèle
  • logs des jobs

🔗 Vroom_relations.php

Gestion des relations entre enregistrements :

  • relations parent → enfants
  • relations inverses
  • pré‑chargement automatique
  • suppression en cascade (optionnelle)
  • cohérence des liens
