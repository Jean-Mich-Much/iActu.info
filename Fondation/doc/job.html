job.php – Gestionnaire de tâches asynchrones
Version 18 – Exécution parallèle & file d’attente

Le module job.php fournit un système léger de gestion de tâches asynchrones pour iActu V18. Il permet d’empiler des appels de fonctions, puis de les exécuter en parallèle via :

  • processus enfants (mode CLI avec pcntl_fork()),
  • processus PHP séparés (mode FPM via job_worker.php).

Il constitue la couche d’orchestration permettant d’exécuter plusieurs parsers RSS en parallèle.

1. Constantes globales

  • JOB_MAX : nombre maximal de jobs simultanés (CLI),
  • JOB_TIMEOUT : durée maximale d’un job (secondes),
  • JOB_LOG_MAX : taille maximale du fichier log,
  • JOB_IS_CLI : vrai si exécuté en ligne de commande.

2. Initialisation

Le module crée automatiquement :

  • le répertoire logs/,
  • le fichier job.log,
  • les variables globales :
  • _JOB_QUEUE : file d’attente des jobs,
  • _JOB_CAPTURE : mode capture actif/inactif,
  • _JOB_LOG : chemin du fichier log.

3. Logging – job_log()

Le fichier job.log est :

  • réinitialisé si > 512 octets,
  • réinitialisé si trop ancien (> 7 jours),
  • alimenté avec des messages horodatés.

4. Début de capture – debut_job()

Active le mode capture :

  • vide la file d’attente,
  • active _JOB_CAPTURE,
  • log « DEBUT JOB ».

Tous les appels suivants via job_call() seront mis en file d’attente.

5. Ajout d’un job – job_call()

Ajoute un job à la file si la capture est active : 

job_call('nom_fonction', ['arg1','arg2']);

Sinon, exécute immédiatement la fonction (mode direct).

Les exceptions sont capturées et loguées.

6. Fin de capture – fin_job()

Désactive la capture, log « FIN JOB », puis déclenche l’exécution de la file via _job_exec().

7. Exécution des jobs – _job_exec()

Deux modes d’exécution :

  • CLI : exécution parallèle via pcntl_fork(),
  • FPM : lancement de processus PHP séparés via job_worker.php.

8. Mode FPM – _job_exec_fpm()

Pour chaque job :

  • construction d’une commande :
php job_worker.php fonction arg1 arg2 ... &
  • exécution en arrière‑plan,
  • pause courte entre chaque lancement.

Ce mode est utilisé lorsque pcntl_fork() n’est pas disponible.

9. Mode CLI – _job_exec_cli()

Exécution parallèle avec gestion fine des processus :

  • jusqu’à JOB_MAX processus simultanés,
  • surveillance des jobs actifs,
  • kill automatique si dépassement de JOB_TIMEOUT,
  • log des succès et erreurs.

Chaque job enfant exécute :

  • call_user_func_array(),
  • log « CHILD OK » ou « EX CHILD ».

10. job_worker.php

Script séparé chargé d’exécuter une fonction dans un processus isolé. Il reçoit :

  • le nom de la fonction,
  • les arguments,
  • et exécute la tâche en environnement indépendant.

11. Résumé opérationnel

  • debut_job() → active la capture
  • job_call() → empile les jobs
  • fin_job() → lance l’exécution
  • _job_exec() → choisit CLI ou FPM
  • _job_exec_cli() → exécution parallèle (fork)
  • _job_exec_fpm() → exécution via job_worker.php
  • job_log() → journalisation

12. Conclusion

Le module job.php fournit un système de tâches asynchrones robuste, léger et parfaitement adapté au pipeline iActu V18. Il garantit :

  • une exécution parallèle contrôlée,
  • une isolation des erreurs,
  • une gestion automatique des timeouts,
  • une compatibilité totale CLI / FPM,
  • une journalisation propre et auto‑rotative.

Il constitue la couche d’orchestration permettant d’exécuter efficacement les parsers RSS en parallèle.
