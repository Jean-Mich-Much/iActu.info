fusion_rss.php – Fusion des flux RSS
Version 18 – Agrégation, déduplication et génération RSS propre

Ce document décrit le fonctionnement du module fusion_rss.php, utilisé par iActu V18 pour fusionner les flux RSS déjà parsés par lit_rss.php.

La fusion permet :

  • d’agréger plusieurs flux d’une même catégorie ;
  • de trier les items par date ;
  • de supprimer les doublons intelligemment ;
  • de générer un flux RSS final propre et valide ;
  • d’assurer la stabilité via logs, validations et écritures atomiques.

1. Rôle général du module

Le module fusion_rss.php intervient après le parsing RSS. Il lit les fichiers XML générés par lit_rss.php, extrait les items, les fusionne, les trie et produit un flux final par catégorie.

Il est utilisé exclusivement par :

  • maj_fusion_cli.php
  • job_worker.php (en mode FPM)

2. Fonction principale : fusion_rss()

La fonction fusion_rss($title, $prefix, $min, $max) effectue la fusion complète d’une catégorie.

2.1 Paramètres

  • $title – nom de la catégorie (ex. “Technologie”) ;
  • $prefix – préfixe des fichiers XML (ex. “tec”) ;
  • $min – nombre minimal d’items requis ;
  • $max – nombre maximal d’items à conserver.

2.2 Étapes principales

  1. création des dossiers cache/rss/fusion/ et logs/ ;
  2. chargement de tous les fichiers XML correspondant au préfixe ;
  3. validation rapide des sources (fusion_valid_src()) ;
  4. extraction des items RSS ;
  5. tri par date décroissante ;
  6. déduplication intelligente (fusion_dedupe()) ;
  7. limitation à $max items ;
  8. validation finale (fusion_valid_final()) ;
  9. écriture atomique du flux final ;
  10. journalisation du résultat.

3. Chargement et validation des sources

Les fichiers source sont récupérés via : 

glob($base.$prefix.'_*.xml')

Chaque fichier est validé par fusion_valid_src() :

  • présence d’au moins un <item> ;
  • présence d’un <title> ;
  • présence des balises </channel> et </rss> ;
  • vérification de la fin du fichier (sécurité anti‑troncature).

Les fichiers invalides sont ignorés.

4. Extraction des items

Pour chaque fichier XML valide :

  • chargement via simplexml_load_string() ;
  • parcours des <item> ;
  • extraction :
    • titre
    • lien
    • date (pubDate → timestamp)
  • filtrage des items incomplets ou trop courts.

5. Tri et déduplication

5.1 Tri chronologique

Les items sont triés du plus récent au plus ancien : 

usort($buffer_flux, fn($a,$b) => $b['date'] <=> $a['date']);

5.2 Déduplication intelligente : fusion_dedupe()

La fonction fusion_dedupe() supprime les doublons selon deux critères :

  • même URL → doublon ;
  • titres similaires → doublon.

En cas de doublon, l’item le plus récent est conservé.

5.3 Détection des titres similaires

La fonction fusion_similar_titles() :

  • normalise les titres (minuscules, nettoyage, espaces) ;
  • extrait les mots significatifs (≥5 lettres) ;
  • compare les mots communs ;
  • considère les titres comme similaires si ≥4 mots identiques.

6. Limitation et validation finale

Après déduplication :

  • les items sont limités à $max ;
  • si le nombre d’items est < $min → fusion abandonnée ;
  • le flux final est généré via fusion_generate_rss() ;
  • le flux est validé via fusion_valid_final().

La validation finale vérifie :

  • présence des items ;
  • présence des titres ;
  • présence des balises </channel> et </rss> ;
  • intégrité de la fin du fichier.

7. Génération du flux final

La fonction fusion_generate_rss() produit un flux RSS 2.0 propre :

  • XML UTF‑8 ;
  • balises <rss> et <channel> ;
  • titre de la catégorie ;
  • items normalisés ;
  • dates au format RFC 822.

L’écriture est atomique : 

$prefix.xml.tmp → rename → $prefix.xml

Cela garantit qu’aucun flux partiellement écrit ne peut être utilisé.

8. Logs de fusion

Les logs sont écrits dans : 

logs/fusion.log

Le fichier est automatiquement réinitialisé si :

  • sa taille dépasse 512 octets ;
  • il date de plus d’une semaine.

Les messages typiques :

  • OK prefix:tec items:32
  • NO XML prefix:vr
  • NO ITEMS prefix:linux
  • NOT ENOUGH ITEMS prefix:apple count:3 min:5
  • INVALID FINAL prefix:jeux

9. Mécanismes de stabilité

Le module intègre plusieurs mécanismes pour garantir la stabilité :

  • validation des sources → évite les flux cassés ;
  • déduplication avancée → évite les doublons multi‑sites ;
  • tri strict → cohérence temporelle ;
  • écriture atomique → aucun flux corrompu ;
  • logs détaillés → diagnostic rapide ;
  • min/max → contrôle du volume final.

10. Conclusion

Le module fusion_rss.php est un agrégateur RSS robuste, conçu pour produire des flux propres, cohérents et fiables à partir d’une grande diversité de sources.

Il garantit :

  • une fusion stable ;
  • une déduplication intelligente ;
  • une validation stricte ;
  • une compatibilité totale avec le pipeline V18 ;
  • une intégration parfaite avec l’affichage HTML.

Il constitue la seconde étape essentielle du système RSS d’iActu.
