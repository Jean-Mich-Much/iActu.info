affiche_html.php – Génération des blocs HTML
Version 18 – Chargement, fallback, tri et affichage des news

Ce document décrit le fonctionnement du module affiche_html.php, utilisé par iActu V18 pour afficher les flux RSS fusionnés sous forme de blocs HTML.

Ce module assure :

  • le chargement sécurisé des flux RSS ;
  • un système de fallback en cas de flux invalide ;
  • la détection automatique du site source ;
  • la génération d’identifiants uniques pour les liens ;
  • le tri des news du jour et des jours précédents ;
  • la production du HTML final (news-box).

1. Rôle général du module

Le module affiche_html.php est utilisé par les pages HTML d’iActu pour afficher les flux fusionnés. Il ne télécharge rien : il lit uniquement les fichiers XML générés par fusion_rss.php.

Il garantit un affichage stable même si un flux est corrompu, vide ou incomplet.

2. Journalisation : log_affiche()

La fonction log_affiche() écrit dans : 

Fondation/logs/affiche_html.log

Elle réinitialise automatiquement le fichier si :

  • sa taille dépasse 512 octets ;
  • il devient trop ancien.

Les logs permettent de diagnostiquer :

  • les flux invalides ;
  • les dates incorrectes ;
  • les fallback utilisés.

3. Chargement des flux : load_flux()

La fonction load_flux($f, $mode) charge un flux RSS depuis :

  • Fondation/cache/rss/ (mode normal)
  • Fondation/cache/rss/fusion/ (mode fusion)

Elle applique une logique de sécurité :

  1. chargement du flux principal ;
  2. validation :
    • XML lisible ;
    • présence d’items ;
    • au moins 5 items.
  3. si valide → copie dans cache/rss/valide/ ;
  4. si invalide → fallback vers la dernière version valide.

En cas d’échec total → false.

4. Détection du site source : news_site_url()

Cette fonction analyse les 5 premiers liens du flux pour déterminer le site principal.

Elle :

  • extrait les URLs ;
  • analyse les domaines ;
  • compte les occurrences ;
  • retourne le domaine dominant (ex. https://www.frandroid.com/).

Si aucun domaine n’est détecté → chaîne vide.

5. Génération d’identifiants uniques

5.1 news_title_id()

Génère un identifiant aléatoire pour le lien du titre :

  • 6 lettres minuscules ;
  • + un nombre entre 0 et 100.

5.2 news_id()

Génère un identifiant unique pour chaque news :

  • année ;
  • jour de l’année ;
  • 7 lettres aléatoires ;
  • + un nombre entre 0 et 1000.

Ces identifiants permettent :

  • le tracking local ;
  • l’unicité des liens ;
  • la compatibilité avec les animations CSS.

6. Fonction principale : affiche()

La fonction affiche($f, $mode, $n) génère un bloc HTML complet pour un flux donné.

6.1 Chargement du flux

Elle appelle load_flux() :

  • si le flux est valide → XML chargé ;
  • si invalide → fallback ;
  • si aucun flux disponible → rien n’est affiché.

6.2 Séparation des news

Les items sont séparés en deux groupes :

  • Aujourd’hui → même date que le jour courant ;
  • Jours précédents → tout le reste.

La fonction :

  • limite les news du jour à $n ;
  • complète avec les jours précédents si nécessaire ;
  • génère les sections correspondantes.

6.3 Génération du HTML

Le HTML produit suit la structure : 

<article class="news-box">
  <header class="news-header">...</header>
  <section class="news-section">...</section>
  ...
</article>

Le bloc contient :

  • le titre du flux ;
  • le lien vers le site source ;
  • le nombre de news du jour ;
  • la liste des news du jour ;
  • la liste des jours précédents ;
  • des messages adaptés (ex. “C’est tout pour aujourd’hui !”).

7. Messages spéciaux

Le module affiche automatiquement :

  • “Rien de neuf ⏰” si aucune news du jour ;
  • “C’est tout pour aujourd’hui !” si toutes les news sont du jour ;
  • “Aujourd’hui : pas de news” si le flux est vide pour la journée.

8. Mécanismes de stabilité

Le module intègre plusieurs mécanismes pour garantir un affichage fiable :

  • fallback automatique vers les flux valides ;
  • logs détaillés en cas d’erreur ;
  • validation minimale (≥5 items) ;
  • identifiants uniques pour éviter les collisions ;
  • séparation jour / anciens jours pour un affichage clair ;
  • aucune dépendance externe → HTML pur.

9. Conclusion

Le module affiche_html.php est responsable de la présentation finale des flux RSS dans iActu V18.

Il garantit :

  • un affichage propre et cohérent ;
  • une tolérance élevée aux flux corrompus ;
  • une expérience utilisateur stable ;
  • une intégration parfaite avec les modules de parsing et de fusion.

Il constitue la dernière étape du pipeline RSS avant l’affichage HTML.
