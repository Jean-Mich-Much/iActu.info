affiche_html.php – Génération HTML des blocs d’actualités
Version 18 – Affichage robuste & fallback automatique

Le module affiche_html.php est responsable de l’affichage final des flux RSS fusionnés dans l’interface iActu V18. Il charge un flux, applique un fallback en cas d’erreur, sépare les news du jour et des jours précédents, puis génère un bloc HTML complet.

1. Journalisation – log_affiche()

Le fichier Fondation/logs/affiche_html.log est :

  • réinitialisé si > 512 octets,
  • alimenté avec des messages horodatés,
  • utilisé pour signaler les flux invalides ou les dates incorrectes.

2. Chargement du flux – load_flux()

Charge un flux RSS depuis :

  • mode fusion : Fondation/cache/rss/fusion/
  • mode normal : Fondation/cache/rss/

Un flux est considéré valide si :

  • le XML se charge correctement,
  • il contient au moins 5 items.

En cas d’échec :

  • le module logue l’erreur,
  • puis charge le flux de fallback dans Fondation/cache/rss/valide/.

Retourne un objet SimpleXML ou false.

3. Détection du site source – news_site_url()

Analyse les 5 premiers liens du flux pour déterminer le domaine principal.

Étapes :

  • extraction des URLs des premiers items,
  • analyse des domaines via parse_url(),
  • comptage des occurrences par domaine,
  • sélection du domaine majoritaire.

Retourne une URL racine (ex : https://www.lemonde.fr/) ou une chaîne vide.

4. Génération d’identifiants – news_title_id() & news_id()

Deux types d’identifiants aléatoires sont utilisés :

  • news_title_id() : ID court pour le lien du titre du site (6 lettres + nombre),
  • news_id() : ID unique pour chaque news (année‑jour + 7 lettres + nombre).

Ces IDs garantissent :

  • l’unicité dans le DOM,
  • la compatibilité avec les animations CSS/JS futures.

5. Affichage HTML – affiche()

La fonction principale affiche($f, $mode, $n) :

  • charge le flux via load_flux(),
  • détecte le site source,
  • sépare les news du jour et des jours précédents,
  • limite le nombre total d’items à $n,
  • génère un bloc HTML complet.

6. Séparation des news

Chaque item est classé selon sa date :

  • Aujourd’hui → même date que date('Y-m-d'),
  • Jours précédents → dates antérieures.

Si les news du jour sont insuffisantes :

  • elles sont complétées par les jours précédents,
  • jusqu’à atteindre $n items.

7. Structure HTML générée

Le module produit un bloc HTML de la forme :

  • <article class="news-box">
  • header avec titre du site + compteur
  • section “Aujourd’hui” (si news)
  • section “C’est tout pour aujourd’hui !” (si aucune news restante)
  • section “Jours précédents” (si applicable)

Chaque news est affichée sous forme de lien : 

<a class="n" id="..." href="...">Titre</a>

8. Gestion des erreurs

Le module logue :

  • les flux invalides,
  • les dates impossibles,
  • les fallbacks utilisés.

En cas d’erreur grave, aucun HTML n’est affiché.

9. Résumé opérationnel

  • load_flux() → charge flux + fallback
  • news_site_url() → détecte le domaine principal
  • news_title_id() / news_id() → IDs uniques
  • affiche() → génère le bloc HTML final
  • log_affiche() → journalisation

10. Conclusion

Le module affiche_html.php fournit un affichage HTML robuste, tolérant et cohérent avec le pipeline iActu V18. Il garantit :

  • un fallback automatique en cas de flux corrompu,
  • une présentation claire et homogène,
  • une séparation fiable des news du jour,
  • des identifiants DOM uniques,
  • une compatibilité totale avec PHP 8.3 / 8.4.

Il constitue la dernière étape du pipeline iActu V18.
