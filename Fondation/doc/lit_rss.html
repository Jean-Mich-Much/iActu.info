lit_rss.php – Lecture et normalisation des flux RSS
Version 18 – Parser robuste, tolérant et auto‑stabilisant

Ce document décrit le fonctionnement complet du module lit_rss.php, utilisé par iActu V18 pour télécharger, analyser, normaliser et stabiliser les flux RSS/Atom. Il s’adresse autant aux débutants qu’aux développeurs expérimentés, en présentant d’abord les concepts simples puis les mécanismes internes avancés.

1. Rôle général du module

Le module lit_rss.php est le cœur du système RSS d’iActu. Il assure :

  • le téléchargement HTTP/HTTPS des flux ;
  • la validation XML (RSS 2.0, Atom, variantes hybrides) ;
  • l’extraction des items (titre, lien, description, date) ;
  • la normalisation complète (UTF‑8, titres, URLs, entités HTML) ;
  • la correction des dates (formats exotiques, dates futures) ;
  • la déduplication intelligente (URL + similarité titres) ;
  • le tri chronologique strict (du plus récent au plus ancien) ;
  • la stabilisation via archives quotidiennes ;
  • la génération d’un flux RSS propre et valide ;
  • la gestion des erreurs et indisponibilités.

Il constitue la première étape du pipeline RSS d’iActu, avant la fusion et l’affichage HTML.

2. Fonction principale : parse()

La fonction parse() possède deux modes :

  • Mode simple (3 arguments) : parse($nom, $url, $max)
    → utilisé pour les flux simples non catégorisés.
  • Mode complet (7 arguments) : parse($titre, $slug, $categorie, $prefixe, $url, $min, $max)
    → utilisé pour les flux iActu (catégories, archives, ready flags, logs, etc.).

Les deux modes suivent la même logique générale :

  1. création du dossier cache ;
  2. création d’un fichier lock ;
  3. vérification du TTL ;
  4. téléchargement du flux ;
  5. parsing XML ;
  6. pipeline de normalisation ;
  7. génération du RSS final ;
  8. validation ;
  9. écriture atomique ;
  10. suppression du lock.

3. Téléchargement du flux : fetch_feed()

Cette fonction télécharge le flux distant avec cURL. Elle gère :

  • user‑agent réaliste (Firefox Linux) ;
  • headers complets (Accept, Accept‑Language, DNT…) ;
  • timeout configurable ;
  • redirections (max 5) ;
  • compression gzip/br ;
  • limite de taille (3 Mo) ;
  • conversion UTF‑8 ;
  • pause aléatoire (25–125 ms) pour éviter les patterns robots.

En cas d’erreur :

  • erreur cURL ;
  • erreur HTTP (≥400) ;
  • flux vide ;
  • flux trop volumineux ;

→ la fonction retourne null et renseigne $rss_last_error.

4. Parsing XML : parse_feed()

Cette fonction :

  • nettoie le XML (clean_xml()) ;
  • charge le flux via simplexml_load_string() ;
  • extrait les items RSS (<item>) ;
  • extrait les items Atom (<entry>) ;
  • retourne un tableau brut : 
    [
  'title' => ...,
  'link'  => ...,
  'desc'  => ...,
  'date'  => timestamp
]

Si le XML est illisible → null.

5. Normalisation des dates

5.1 normalize_date()

Cherche dans l’ordre :

  • pubDate
  • published
  • updated
  • date

Puis appelle try_parse_date().

5.2 try_parse_date()

Essaye :

  • strtotime()
  • formats manuels (RFC, ISO, exotiques)

Puis corrige les dates futures via fix_future_date().

6. Pipeline de traitement des items

6.1 Tri brut : sort_items()

Tri décroissant par date : 

usort($it, fn($a,$b) => $b['date'] <=> $a['date']);

6.2 Normalisation : normalize_items()

Pour chaque item :

  • titre → normalize_title()
  • URL → normalize_url()
  • validation URL
  • suppression des items invalides

6.3 Filtrage : filter_items()

Supprime les items trop courts :

  • titre < 5 caractères
  • lien < 5 caractères

6.4 Déduplication : dedupe_items()

Déduplication intelligente :

  • même URL → doublon
  • titres similaires → doublon
  • si doublon → on garde l’item le plus récent

6.5 Tri final + slice

Tri final par date puis : 

$buffer = array_slice($buffer, 0, $max);

7. Génération du RSS final : generate_rss()

Produit un flux RSS 2.0 propre :

  • XML UTF‑8
  • <rss>, <channel>
  • titre + catégorie
  • items normalisés
  • dates au format RFC 822

8. Validation du flux : validate_buffer()

Vérifie :

  • nombre d’items exact
  • présence des titres
  • présence des liens
  • présence des dates
  • présence des balises RSS/Channel
  • XML valide

En cas d’erreur → false + message dans $rss_last_error.

9. Gestion des archives quotidiennes

9.1 rss_save_daily()

Sauvegarde brute du flux du jour :

  • 1 fichier par jour
  • nom : prefixe_slug_JJ.xml

9.2 rss_load_daily()

Charge jusqu’à 31 jours d’archives, triées par date de fichier.

9.3 rss_cleanup_daily()

Supprime les archives au‑delà de 31 jours.

10. Gestion des indisponibilités : rss_log_indispo()

Crée un log par flux :

  • erreurs HTTP
  • erreurs XML
  • flux vide
  • flux illisible
  • archives trop anciennes (>4h)

11. Mécanismes de stabilité

Le parser intègre plusieurs mécanismes pour garantir la stabilité :

  • TTL aléatoire (30–60 min) → évite les rafales
  • lock fichier → empêche les exécutions simultanées
  • ready flag → indique qu’une archive est utilisable
  • fallback automatique sur archives
  • tri strict par date
  • déduplication intelligente
  • limitation stricte du nombre d’items
  • écriture atomique via fichier .tmp

12. Conclusion

Le module lit_rss.php est un parser RSS complet, robuste et tolérant, conçu pour absorber la diversité des flux du web moderne. Il garantit :

  • une extraction propre ;
  • une normalisation cohérente ;
  • une stabilité dans le temps ;
  • une résilience face aux flux cassés ;
  • une intégration parfaite avec le système iActu V18.

Il constitue la base fiable sur laquelle reposent la fusion, l’affichage HTML et l’expérience utilisateur d’iActu.
