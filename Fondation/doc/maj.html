maj_cli.php & maj.php – Orchestration du pipeline RSS
Version 18 – Mise à jour automatique & exécution parallèle

Les fichiers maj_cli.php et maj.php constituent la couche d’orchestration du pipeline iActu V18. Ils déclenchent l’ensemble des parsers RSS, puis les fusions, en utilisant le système de jobs asynchrones fourni par job.php.

1. maj_cli.php – Exécution complète du pipeline

Ce script est le cœur de la mise à jour RSS. Il est conçu pour être exécuté :

  • en ligne de commande (cron),
  • ou indirectement via maj.php.

Il charge les modules essentiels :

  • job.php – gestion des jobs asynchrones,
  • lit_rss.php – parsing individuel des flux,
  • fusion_rss.php – fusion des flux par catégories.

2. Structure générale

Le script suit une structure simple :

  1. debut_job() – activation de la capture des jobs,
  2. job_call() – empilement de tous les parsers RSS,
  3. job_call() – empilement des fusions finales,
  4. fin_job() – exécution parallèle de tous les jobs.

Chaque section correspond à une catégorie du site :

  • Technologie
  • Tech. Autres
  • Tech. English
  • Apple / Apple autres / Apple english
  • Jeux / Jeux autres / Jeux english
  • Linux
  • Réalité virtuelle
  • Infos France
  • Infos Monde
  • Sciences / Sciences autres / Sciences english
  • Insolites
  • Cinéma TV
  • Sports
  • Infos English

Chaque flux est traité via : 

job_call('parse', [titre, prefixe, catégorie, dossier, url, min, max]);

Puis fusionné via : 

job_call('fusion_rss', [titre, prefixe, min, max]);

3. Rôle des paramètres parse()

Chaque appel parse() reçoit :

  • Nom du site (affichage),
  • Préfixe (nom des fichiers XML),
  • Catégorie (affichage),
  • Dossier (ex : tec, jeux, infos_fr),
  • URL du flux RSS,
  • min : nombre minimal d’items valides,
  • max : nombre maximal d’items conservés.

Le parser génère un fichier : 

cache/rss/prefixe_XXXX.xml

…ainsi qu’un fichier .ready indiquant que le flux est prêt pour la fusion.

4. Rôle des fusions – fusion_rss()

Chaque catégorie est fusionnée via : 

job_call('fusion_rss', [titre, prefixe, min, max]);

La fusion :

  • collecte tous les flux prefixe_*.xml,
  • attend les fichiers .ready,
  • valide les flux sources,
  • déduplique les items (URL + 3 mots ≥ 5 lettres),
  • génère un flux final dans :
cache/rss/fusion/prefixe.xml

5. Exécution parallèle – job.php

Une fois tous les jobs empilés, fin_job() déclenche :

  • mode CLI → pcntl_fork() (parallélisme réel),
  • mode FPM → job_worker.php (processus séparés).

Le système gère :

  • les timeouts,
  • les erreurs isolées,
  • la rotation automatique des logs.

6. maj.php – Interface HTTP

Le fichier maj.php permet de déclencher la mise à jour via HTTP. Il ne lance jamais le pipeline directement : il démarre maj_cli.php en arrière‑plan.

Fonctionnement :

  • si exécuté en CLI → inclut directement maj_cli.php,
  • si exécuté via HTTP → lance :
php maj_cli.php > /dev/null 2>&1 &

Puis renvoie immédiatement :

HTTP 204 No Content

Cela garantit :

  • aucun blocage du navigateur,
  • aucune attente,
  • une mise à jour totalement asynchrone.

7. Résumé opérationnel

  • maj.php → déclencheur HTTP (asynchrone)
  • maj_cli.php → orchestration complète du pipeline
  • parse() → parsing individuel des flux
  • fusion_rss() → fusion par catégories
  • job.php → exécution parallèle & gestion des erreurs

8. Conclusion

Le duo maj.php / maj_cli.php constitue la couche d’orchestration du système iActu V18. Il garantit :

  • une mise à jour fiable et entièrement automatisée,
  • une exécution parallèle optimisée,
  • une tolérance totale aux erreurs individuelles,
  • une compatibilité CLI / HTTP parfaite,
  • une intégration fluide avec les modules lit_rss, fusion_rss et affiche_html.

Il s’agit de la pièce maîtresse du pipeline RSS d’iActu V18.
