Système de mise à jour RSS
Version 18 – Parsers, fusions, stabilité et exécution automatique

Ce document décrit le fonctionnement complet du système de mise à jour d’iActu V18. Il couvre les quatre scripts principaux :

  • maj.php – Lanceur HTTP des parsers RSS ;
  • maj_cli.php – Exécution CLI des parsers RSS ;
  • maj_fusion.php – Lanceur HTTP des fusions RSS ;
  • maj_fusion_cli.php – Exécution CLI des fusions RSS.

Ces scripts sont déclenchés automatiquement par les crontabs toutes les 10 minutes, et peuvent également être lancés en fallback par les pages HTML si les crons ne se sont pas exécutés.

1. Rôle général du système de mise à jour

Le système de mise à jour V18 assure :

  • le téléchargement et parsing de plus de 150 flux RSS/Atom ;
  • la fusion des flux par catégories (Technologie, Jeux, Apple, Sciences…) ;
  • la génération des pages HTML quotidiennes ;
  • la gestion des caches, locks, tempos et TTL ;
  • la stabilité globale du système même en cas de surcharge ou d’erreurs réseau.

Il fonctionne en deux étapes :

  1. maj_cli.php → télécharge et normalise tous les flux RSS ;
  2. maj_fusion_cli.php → fusionne les flux par catégories.

Les versions HTTP (maj.php et maj_fusion.php) ne font que lancer les versions CLI en arrière‑plan.

2. maj.php – Lanceur HTTP des parsers RSS

Le script maj.php est appelé par les pages HTML lorsque :

  • le cache est trop ancien ;
  • le fichier du jour n’existe pas ;
  • le système détecte un besoin de rafraîchissement forcé.

Son rôle est simple :

  • si exécuté en HTTP → lance maj_cli.php en tâche de fond ;
  • si exécuté en CLI → délègue directement à maj_cli.php.

Il renvoie systématiquement un code HTTP 204 (no content).

3. maj_cli.php – Exécution CLI des parsers RSS

Le script maj_cli.php est le cœur du système de mise à jour. Il exécute l’ensemble des parsers RSS définis dans la configuration.

3.1 Initialisation

  • création d’un fichier maj_tempo ;
  • chargement des modules :
    • job.php – gestion des jobs ;
    • lit_rss.php – parser RSS V18 ;
    • fusion_rss.php – fusion interne (utilisée par certains jobs).
  • génération d’un $max aléatoire (20–40 items).

3.2 Construction des jobs

Le script construit un tableau de plus de 150 jobs RSS, répartis par catégories :

  • Technologie (principaux + autres + english) ;
  • Apple (FR + autres + english) ;
  • Jeux (FR + autres + english) ;
  • Linux ;
  • Réalité virtuelle ;
  • Infos France ;
  • Infos Monde ;
  • Sciences (FR + autres + english) ;
  • Insolites ;
  • Cinéma/TV ;
  • Sports ;
  • Infos English.

Chaque job appelle la fonction : 

parse($titre, $slug, $categorie, $prefixe, $url, $min, $max)

3.3 Exécution

  • mélange aléatoire des jobs (shuffle()) ;
  • exécution séquentielle via job_call() ;
  • gestion automatique des erreurs, TTL, locks et écritures atomiques.

4. maj_fusion.php – Lanceur HTTP des fusions RSS

Le script maj_fusion.php fonctionne exactement comme maj.php, mais pour les fusions RSS.

Il lance en arrière‑plan : 

maj_fusion_cli.php

et renvoie un code HTTP 204.

5. maj_fusion_cli.php – Fusion des flux par catégories

Le script maj_fusion_cli.php fusionne les flux déjà parsés par maj_cli.php.

5.1 Initialisation

  • création du fichier maj_fusion_tempo ;
  • pause aléatoire (15–25 secondes) pour éviter les collisions ;
  • chargement des modules :
    • job.php
    • fusion_rss.php
  • génération d’un $max aléatoire (20–40 items).

5.2 Liste des fusions

Chaque fusion regroupe les flux d’une catégorie :

  • Réalité virtuelle
  • Linux
  • Technologie (FR / autres / english)
  • Apple (FR / autres / english)
  • Jeux (FR / autres / english)
  • Sciences (FR / autres / english)
  • Infos France
  • Infos Monde
  • Insolites
  • Cinéma / TV
  • Sports
  • Infos English

Chaque job appelle : 

fusion_rss($titre, $prefixe, $min, $max)

5.3 Exécution

  • mélange aléatoire (shuffle()) ;
  • exécution via job_call('fusion_rss') ;
  • génération des flux fusionnés prêts pour l’affichage HTML.

6. Intégration avec les pages HTML

Les pages HTML d’iActu peuvent déclencher automatiquement les mises à jour si :

  • le cache est trop ancien ;
  • le fichier du jour n’existe pas ;
  • le contenu généré est incomplet ;
  • les crons ne se sont pas exécutés.

Le mécanisme repose sur :

  • minute_pair – rafraîchissement toutes les 2 minutes ;
  • lock – empêche les exécutions simultanées ;
  • tempo_maj et tempo_fusion – évitent les rafales ;
  • exec("php maj.php &") – lancement en arrière‑plan.

Les pages HTML valident également :

  • la présence du DOCTYPE ;
  • la présence des blocs news-box ;
  • la présence des blocs class="n" ;
  • la taille minimale du contenu.

7. Mécanismes de stabilité

Le système de mise à jour V18 intègre plusieurs mécanismes de stabilité :

  • tempo – limite la fréquence d’exécution ;
  • locks – évite les exécutions concurrentes ;
  • shuffle – répartit la charge ;
  • sleep aléatoire – évite les collisions ;
  • écriture atomique – fichiers .tmp puis rename ;
  • fallback automatique via les pages HTML ;
  • TTL – rafraîchissement contrôlé des flux.

8. Conclusion

Le système de mise à jour d’iActu V18 est conçu pour être :

  • robuste ;
  • tolérant aux erreurs ;
  • auto‑stabilisant ;
  • compatible avec les crons et le fallback HTTP ;
  • capable de traiter un grand nombre de flux sans surcharge.

Il constitue l’infrastructure essentielle sur laquelle reposent les pages HTML quotidiennes d’iActu.
