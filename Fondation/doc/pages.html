pages.php – Architecture des pages dynamiques
Version 18 – Cache intelligent & mise à jour automatique

Les fichiers Page_technologie.php, Page_Apple.php, Page_jeux.php, etc. constituent les pages principales du site iActu V18. Chaque page suit la même architecture : cache local, fallback, affichage HTML, et déclenchement automatique de la mise à jour RSS.

1. Variables principales

Chaque page définit :

  • $titre_page : titre global du site,
  • $nom_page : préfixe des fichiers HTML mis en cache,
  • $cache_secondes : durée maximale du cache,
  • $caracteres : taille minimale du HTML pour validation,
  • $page_active : identifiant du menu actif,
  • $sites : nom du fichier de contenu (ex : technologie.php).

Ces variables permettent de réutiliser la même structure pour toutes les catégories.

2. Système de cache – double fichier

Chaque page utilise un cache basé sur deux fichiers :

  • Page_xxx_jour_.html → cache principal,
  • Page_xxx_jour_<jour>.html → archive du jour.

Le cache est utilisé si :

  • le fichier existe,
  • il est récent ($cache_secondes),
  • le fichier du jour existe.

Sinon, la page est régénérée.

3. Rafraîchissement toutes les minutes paires

Le système alterne :

  • minutes impaires → lecture du cache,
  • minutes paires → possibilité de régénération.

Cela réduit la charge serveur tout en gardant un site très réactif.

4. Génération du HTML

Lorsque la page doit être régénérée :

  1. activation d’un buffer (ob_start()),
  2. chargement du module affiche_html.php,
  3. construction du HTML complet :
  • header + CSS,
  • menu haut,
  • messages haut,
  • contenu du site (Fondation/sites/$sites.php),
  • messages bas,
  • menu bas.

Le HTML est ensuite validé :

  • présence d’un seul <!DOCTYPE html>,
  • présence d’un seul </html>,
  • taille minimale ($caracteres).

Si valide :

  • affichage immédiat,
  • écriture dans $nom_page.html,
  • copie dans $nom_page<jour>.html.

Sinon → fallback sur le cache existant.

5. Déclenchement automatique de la mise à jour RSS

Après l’affichage, la page peut déclencher une mise à jour RSS via maj.php.

Conditions :

  • minute impaire ou première visite (pas de cache),
  • fichier maj_lock absent ou trop ancien (> 60 s).

Si les conditions sont réunies : 

exec("php maj.php > /dev/null 2>&1 &");

Cela lance la mise à jour en arrière‑plan, sans bloquer l’affichage.

6. Rôle du fichier maj_lock

Le fichier maj_lock empêche les mises à jour trop fréquentes. Il est recréé à chaque lancement de maj.php.

Il expire automatiquement après 60 secondes.

7. Résumé opérationnel

  • minutes impaires → lecture du cache
  • minutes paires → possibilité de régénération
  • validation HTML → sécurité anti‑corruption
  • cache du jour → stabilité visuelle
  • maj_lock → anti‑spam de mise à jour
  • maj.php → mise à jour RSS asynchrone

8. Conclusion

Les pages dynamiques d’iActu V18 utilisent un système de cache intelligent, une validation stricte du HTML et un déclenchement automatique de la mise à jour RSS. Elles garantissent :

  • une vitesse d’affichage optimale,
  • une stabilité visuelle,
  • une mise à jour continue des flux RSS,
  • une charge serveur minimale.

Ce mécanisme constitue la couche finale du pipeline iActu V18.
