Pages de news iActu
Version 18 – Cache, fallback, affichage et mise à jour automatique

Ce document décrit le fonctionnement des cinq pages principales d’iActu V18 : Page_technologie.php, Page_Apple.php, Page_jeux.php, Page_sciences.php et Page_actualités.php.

Ces pages constituent la dernière étape du pipeline RSS : elles affichent les flux fusionnés, gèrent le cache, déclenchent les mises à jour et produisent les pages HTML statiques visibles par les visiteurs.

1. Rôle général des pages

Chaque page :

  • charge les flux fusionnés ou non fusionnés ;
  • génère le HTML final via affiche_html.php ;
  • met en cache la page générée ;
  • déclenche les mises à jour si nécessaire ;
  • affiche les menus, messages et sections CSS ;
  • gère les pages du jour précédent / suivant.

Les pages sont statiques pour les visiteurs, mais dynamiques en interne grâce au système de cache.

2. Variables principales

Chaque page définit plusieurs variables essentielles :

  • $titre_page – titre global (ex. “iActu”) ;
  • $nom_page – préfixe des fichiers HTML du jour ;
  • $page_active – identifiant pour le menu ;
  • $sites – nom du fichier dans Fondation/sites/ ;
  • $cache_secondes – durée maximale du cache ;
  • $caracteres – taille minimale du HTML généré ;
  • $news_box – nombre minimal de blocs news-box ;
  • $class_n – nombre minimal de liens class="n".

Ces valeurs garantissent que la page générée est complète et cohérente.

3. Gestion du cache

Chaque page génère deux fichiers :

  • Page_xxx.html – cache principal ;
  • Page_xxxJJ.html – archive du jour (JJ = jour du mois).

Le système vérifie :

  • si les fichiers existent ;
  • si le cache est trop ancien ;
  • si le contenu généré est valide.

Si tout est correct → la page HTML est simplement lue et affichée. Sinon → une nouvelle version est générée.

4. minute_pair – Rafraîchissement toutes les 2 minutes

Le système alterne entre :

  • minute paire → génération possible ;
  • minute impaire → lecture du cache uniquement.

Cela réduit la charge serveur et évite les rafales de génération.

5. Génération du HTML

Lorsque la page doit être régénérée :

  1. un buffer est ouvert (ob_start()) ;
  2. affiche_html.php est chargé ;
  3. le squelette HTML est généré :
    • header
    • CSS (fonts, base, menu, messages, mid)
    • menu haut
    • messages top
    • contenu principal (via Fondation/sites/xxx.php)
    • messages bas
    • menu bas
  4. le contenu est validé ;
  5. si valide → écrit dans le cache ;
  6. sinon → fallback vers l’ancienne version.

6. Validation du contenu généré

Le HTML généré doit respecter plusieurs critères :

  • présence d’un seul <!DOCTYPE html> ;
  • présence d’un seul </html> ;
  • au moins $news_box blocs class="news-box" ;
  • au moins $class_n liens class="n" ;
  • taille minimale $caracteres.

Si un critère échoue → fallback vers le cache précédent.

7. Déclenchement automatique des mises à jour

Les pages peuvent déclencher :

  • maj.php → parsers RSS ;
  • maj_fusion.php → fusions RSS.

Conditions :

  • minute paire ;
  • ou $force_maj si les fichiers du jour n’existent pas ;
  • lock non présent ou trop ancien ;
  • tempo_maj / tempo_fusion dépassés.

Les mises à jour sont lancées en arrière‑plan : 

exec("php maj.php > /dev/null 2>&1 &");

8. Contenu principal : Fondation/sites/*.php

Chaque page inclut un fichier dans Fondation/sites/ correspondant à sa catégorie.

Exemple pour la technologie : 

Fondation/sites/technologie.php

Ce fichier contient une grille de blocs :

  • flux individuels (non fusionnés) ;
  • flux fusionnés (vr, linux, tecautres, tecenglish).

Chaque bloc est généré via : 

affiche('prefixe_flux', 'mode', nombre_items)

Exemples :

  • affiche('tec_clubic','pasfusion',15)
  • affiche('vr','fusion',15)

9. Menus et navigation

Les pages incluent :

  • menu.php – navigation principale ;
  • messages_top.php – messages d’information ;
  • messages_bot.php – messages bas de page ;
  • menu_bot.php – navigation bas de page.

9.1 menu.php

Le menu contient :

  • logo iACTU ;
  • liens vers les 5 pages ;
  • date du jour ;
  • navigation jour précédent / suivant ;
  • heure de dernière mise à jour.

9.2 messages_top.php / messages_bot.php

Ces fichiers affichent des messages optionnels (dons, annonces, etc.). Ils sont entièrement configurables.

10. CSS des pages

Le design complet est documenté ici :

  • Documentation CSS iActu

Résumé rapide des fichiers CSS utilisés :

  • fonts.css – polices et variables ;
  • base.css – structure générale, couleurs, resets ;
  • menu.css – navigation ;
  • messages.css – messages top/bot ;
  • mid.css – grille des news, responsive, spacing.

Le CSS est minimaliste, rapide, optimisé pour mobile et desktop.

11. Mécanismes de stabilité

Les pages intègrent plusieurs mécanismes pour garantir un affichage fiable :

  • cache statique → rapidité maximale ;
  • fallback automatique → jamais de page cassée ;
  • validation stricte du HTML généré ;
  • minute_pair → génération contrôlée ;
  • locks → pas de double exécution ;
  • tempo → évite les rafales de mises à jour.

12. Conclusion

Les pages de news d’iActu V18 constituent la dernière étape du pipeline RSS. Elles assurent :

  • un affichage propre et rapide ;
  • une mise à jour automatique fiable ;
  • une tolérance élevée aux erreurs ;
  • une navigation fluide entre les jours ;
  • une intégration parfaite avec les modules internes.

Elles sont la vitrine finale d’iActu, combinant performance, stabilité et simplicité.
