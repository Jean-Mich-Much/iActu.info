iActu V18 – Documentation CSS
Design System complet & Architecture proportionnelle

Le système CSS d’iActu V18 repose sur une architecture unifiée, indépendante des OS, entièrement proportionnelle en mode HD, et conçue pour garantir une cohérence psychovisuelle parfaite sur tous les écrans.

1. Philosophie générale

Le CSS d’iActu V18 suit quatre principes fondamentaux :

  • Indépendance totale vis‑à‑vis des polices système et des emojis,
  • Proportionnalité automatique pour les écrans ≥ 1921px,
  • Psychovisuel constant : même rendu sur tous les OS et navigateurs,
  • Minimalisme fonctionnel : aucune propriété inutile, aucune surcharge.

Les polices, emojis et métriques sont embarqués localement, garantissant une cohérence absolue.

2. Architecture des fichiers CSS

  • base.css → fondations, couleurs, typographie, structure globale
  • mid.css → affichage des blocs d’actualités
  • menu.css → navigation principale
  • messages.css → messages haut/bas
  • fonts.css → polices embarquées (Inter + Twemoji)

Chaque fichier est autonome mais cohérent grâce aux variables globales définies dans :root.

3. Variables globales – :root

Définies dans base.css, elles constituent le cœur du design system :

  • Palette sombre (c1 → c10),
  • Couleurs d’accent (jaune, bleu, orange),
  • Rayons arrondis (pill),
  • Version proportionnelle via round(calc(...)).

Exemple : 

--c1:rgba(12, 10, 14, 1);
--radius-pill:999px;
--radius-pill-p:round(calc(999/1920*100vw),1px);

Toutes les unités critiques (padding, border, gap, font-size) existent en version px et version proportionnelle.

4. Indépendance typographique – fonts.css

iActu V18 ne dépend d’aucune police système. Les polices sont chargées depuis : 

Fondation/css/fonts.css

Elles incluent :

  • Inter (texte principal),
  • Twemoji (emoji vectoriels),
  • Fallbacks internes uniquement.

Avantages :

  • rendu identique sur Windows, macOS, Linux, Android, iOS,
  • aucune variation de graisse ou d’interlignage,
  • aucune dépendance aux polices natives.

5. Base CSS – base.css

Le fichier base.css définit :

  • le reset minimal,
  • la typographie globale,
  • la structure flex verticale,
  • les couleurs de fond,
  • les bordures haut/bas des zones top/bot/messages.

Tous les éléments partagent :

  • font-family: "Inter", sans-serif, "Twemoji",
  • font-size: 16px (ou proportionnel en HD),
  • line-height: 22px,
  • hyphens:auto,
  • text-rendering: optimizeLegibility.

Le mode HD (≥ 1921px) convertit automatiquement :

  • font-size,
  • line-height,
  • border-width,
  • padding,
  • gap,
  • radius.

6. Composants d’affichage – mid.css

Le fichier mid.css gère l’affichage des blocs d’actualités :

  • grille responsive (2 colonnes → 1 colonne < 1152px),
  • cartes (news-box),
  • titres de sites (news-title),
  • compteurs (news-count),
  • sections (Aujourd’hui / Jours précédents),
  • liste des news.

Chaque élément utilise :

  • couleurs issues de :root,
  • rayons arrondis cohérents,
  • bordures fines (1px ou proportionnelles),
  • espacements constants (gap, padding).

Le responsive repose sur deux seuils :

  • ≤ 1152px → 1 colonne, texte wrap, liens multi‑lignes,
  • ≥ 1921px → mode HD proportionnel.

7. Menu principal – menu.css

Le fichier menu.css gère la navigation :

  • menu centré,
  • icônes emoji Twemoji,
  • effet de sélection (outline + skew + rotation),
  • mode compact < 1152px (icônes seules),
  • mode HD proportionnel.

Le menu actif utilise :

  • outline jaune (c6),
  • rotation 10°,
  • skew -10° / -10°,
  • accentuation de la graisse.

8. Messages haut/bas – messages.css

Les messages utilisent :

  • fond c1,
  • bordures fines (haut/bas),
  • texte wrap automatique,
  • emoji Twemoji,
  • mode HD proportionnel.

Le sélecteur spécial .coucou génère un format e‑mail inversé (RTL) pour éviter les robots.

9. Psychovisuel : cohérence totale

Le système CSS garantit :

  • aucune variation de rendu entre navigateurs,
  • aucune dépendance aux polices natives,
  • aucune différence entre Windows/macOS/Linux/iOS/Android,
  • une grille stable, même en zoom,
  • un rendu identique en 1080p, 1440p, 4K, 5K.

Le mode HD proportionnel assure que :

  • les proportions restent exactes,
  • les arrondis restent harmonieux,
  • les bordures restent visibles mais fines,
  • les espacements restent psychovisuellement constants.

10. Résumé opérationnel

  • base.css → fondations, couleurs, typographie, structure
  • mid.css → affichage des blocs d’actualités
  • menu.css → navigation
  • messages.css → messages haut/bas
  • fonts.css → polices embarquées (Inter + Twemoji)
  • HD mode → proportionnel automatique ≥ 1921px

11. Conclusion

Le système CSS d’iActu V18 est conçu pour offrir :

  • une cohérence visuelle absolue,
  • une indépendance totale vis‑à‑vis des OS,
  • un rendu psychovisuel stable,
  • une lisibilité optimale,
  • une esthétique moderne et minimaliste.

Il constitue l’un des piliers de l’identité visuelle d’iActu V18.
